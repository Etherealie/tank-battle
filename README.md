# 坦克大战 (Tank Battle)

经典红白机坦克大战的网页复刻版，纯 HTML/CSS/JavaScript 实现，单文件即开即玩。

## 快速开始

双击 `index.html`，用任意现代浏览器打开即可。无需安装任何依赖，无需构建工具。

- **↑ ↓ ← → 或 WASD**：移动坦克
- **Space**：开火
- **P**：暂停 / 继续
- **Enter**：开始游戏 / 下一关 / 重新开始

## 构建过程

> 以下记录了这个项目从零到一的完整开发思路和实现细节。

### 一、技术选型：为什么是纯 HTML 单文件？{ #section--}

这个项目的核心诉求是 **零门槛可玩**，放在任何一个有浏览器的设备上双击就能跑。因此排除了所有需要构建工具、包管理器、服务端环境的方案。

最终选择 **单个 HTML 文件内嵌 CSS + JavaScript**，所有代码自包含。Canvas API 作为渲染层，因为它是浏览器原生提供的 2D 绘图能力，不需要 DOM 操作和 CSS 动画，对游戏这种高频重绘场景比 DOM 方案高效得多。

### 二、架构设计：概念模型到代码映射{ #section-二}

在写第一行代码之前，先在脑中建模——把"坦克大战"拆解成几个独立的概念：

| 概念 | 代码实体 | 职责 |
|---|---|---|
| 地图 | `map[][]` 二维数组 | 存储每个格子的类型（空地/砖墙/钢墙/水） |
| 坦克 | `class Tank` | 位置、方向、速度、碰撞检测、射击冷却 |
| 子弹 | `class Bullet` | 飞行方向、速度、碰撞判定 |
| 爆炸 | `class Explosion` | 纯视觉效果，生命周期衰减 |
| 游戏状态机 | `gameState` 变量 | idle → playing → paused / dying → over / levelup |

这种拆法遵循一个朴素原则：**每个类只做一件事，类的实例之间通过碰撞检测产生交互**，而不是让一个上帝类去管理一切。

### 三、坐标系统与碰撞检测{ #section-三}

地图采用 **瓦片网格**（Tile Grid）：画布 780×600，每个瓦片 30×30，共 26 列 × 20 行。坦克坐标用画布像素值，需要碰撞检测时转成包围盒（bounding box）。

```
瓦片坐标 ←→ 像素坐标
  col = floor(pixel_x / 30)
  row = floor(pixel_y / 30)
```

碰撞检测分两层：

1. **坦克 vs 地图**：取坦克包围盒四个角对应的瓦片，检查是否有不可通行的类型
2. **坦克 vs 坦克**：AABB 矩形相交判定，没有任何宽恕余量
3. **子弹 vs 坦克**：子弹看作一个点，用点是否落在坦克包围盒内的判定
4. **子弹 vs 地图**：子弹落点对应的瓦片，钢墙直接消失，砖墙消失且破坏瓦片

### 四、敌人 AI：简单但有效的设计{ #section-四}

敌人的行为逻辑非常短，核心就几行：

```
1. 2% 概率随机转向
2. 沿当前方向移动
3. 如果撞墙/撞坦克/出界 → 回退并随机转向
4. 2% 概率开火（受冷却时间限制）
```

没有寻路算法，没有 A*，没有行为树。为什么够用？

因为关卡地图本身就是迷宫结构，随机转向 + 碰撞回退的机制自然会产生"在通道里来回巡逻"的行为。加上多个敌人同时在场，形成的交叉火力网已经足够有压迫感。**复杂不等于好玩**，这是一种刻意保持的设计克制。

### 五、渲染层：Canvas 绘制的细节{ #section-五}

每帧的绘制顺序是有讲究的：

1. **背景网格** → 最底层，纯装饰
2. **地图瓦片** → 砖墙画砖纹、钢墙画金属渐变、水画正弦波动态
3. **子弹** → 带发光效果的小圆点
4. **敌人坦克** → 车身 + 履带 + 炮管 + 炮塔，Canvas 旋转实现朝向
5. **玩家坦克** → 同上，颜色区分（绿色 vs 红色系）
6. **爆炸粒子** → 径向渐变，生命周期内从亮到暗、从小到大

坦克的旋转利用 `ctx.rotate()`，只画朝上的坦克，然后旋转画布。这样做的好处是只需要一套绘制逻辑，四个方向都能复用。

### 六、游戏状态机{ #section-六}

```
  [idle] ──Enter──→ [playing]
    ↑                  │
    │        ┌─────────┼─────────┐
    │        ↓         ↓         ↓
    │     [paused]  [dying]  [all enemies dead]
    │        │         │         │
    │        P         │     计时60帧
    │        │         │         ↓
    │        ↓         ↓     [levelup]
    │     [playing]  [over]      │
    │                  │      Enter
    └──Enter───────────┴─────────┘
```

状态机的好处：不同状态下运行的逻辑完全隔离。暂停时不跑 update()，死亡动画期间不接受操作，关卡完成时有过渡计时防止秒过。

### 七、一些有趣的实现细节{ #section-七}

- **无敌帧闪烁**：`invincible % 4 < 2` 条件跳过绘制，产生每 2 帧可见、2 帧不可见的闪烁效果，比控制透明度更接近原版手感
- **子弹冷却**：玩家冷却 20 帧，敌人冷却随关卡递减（60 - level×3），后期敌人射速明显提升
- **出生点保护**：玩家出生时 60 帧无敌，死亡重生后 120 帧无敌，防止连续被秒
- **敌人出生碰撞检测**：敌人生成前检查是否与其他坦克重叠，避免穿模
- **瓦片破坏**：子弹击中砖墙时不只是删除该瓦片，还会生成一个小爆炸粒子
